# CT Report

Веб-сервис для интеллектуальной генерации заключения врача-кардиохирурга по данным КТ аорты.

## О проекте

**CT Report** — это веб-сервис, который принимает от пользователя набор изображений (срезы КТ аорты) и таблицу измерений (диаметры/площади/периметры по срезам), а затем автоматически формирует структурированный медицинский отчёт в виде PDF и HTML по заданному шаблону. Генерация текста отчета выполняется LLM, качество отчета оценивается пользователем.

## Ключевые возможности

### Для пользователя
- **Загрузка кейса** — изображения (PNG/JPEG/ZIP), измерения (CSV/JSON), метаданные пациента (ФИО/шифр, возраст, пол, дата исследования, анамнез).
- **Валидация данных** — проверка структуры измерений, единиц измерения, диапазонов значений.
- **Просмотр отчётов** — список сгенерированных заключений, онлайн-рендер HTML.
- **Экспорт отчёта** — скачивание итогового документа в формате PDF.
- **Оценка качества** — рейтинг 1–5 + текстовый комментарий об ошибках/неточностях.

### Для администратора
- **Управление шаблонами** — версионирование промптов и структур отчётов.
- **Метрики качества** — средняя оценка, частота ошибок валидации, время генерации.
- **Tracing & Observability** — логирование всех LLM-вызовов.
- **Клинические протоколы** — загрузка и обновление справочных медицинских документов.
## Пользовательский сценарий

1. **Загрузка** Пользователь отправляет изображения, таблицу измерений и метаданные.
2. **Валидация** Система проверяет формат, единицы измерения и допустимые диапазоны.
3. **Контекст** Из базы клинических протоколов подгружается релевантная справочная информация.
4. **Генерация** LLM формирует блоки: диагноз/описание, таблица ключевых размеров, выводы, рекомендации.
5. **Экспорт** Рендерится HTML и PDF по утверждённому медицинскому шаблону.
6. **Оценка** Врач оценивает отчёт, данные сохраняются для аудита и дообучения.
7. **Архив** В БД фиксируются исходные файлы, извлечённые измерения, версия промпта/модели, трассировки и итоговые отчёты.
