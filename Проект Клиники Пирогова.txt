Проект: веб-сервис для интеллектуальной генерации заключения врача-кардиохирурга по данным КТ

Задача: Разработать веб-сервис, который принимает от пользователя набор изображений (срезы КТ аорты) и таблицу измерений (диаметры/площади/периметры по срезам), а затем автоматически формирует структурированный медицинский отчёт в виде PDF и/или HTML по заданному шаблону. Генерация текста отчёта выполняется LLM, качество отчёта оценивается пользователем.

Пользовательский сценарий: 
1) Пользователь загружает кейс: изображения (PNG/JPEG, возможно ZIP) + измерения (CSV/JSON) + базовые метаданные пациента (ФИО/шифр, возраст, пол, дата исследования, анамнез).
2) Сервис валидирует входные данные (структура измерений, единицы, диапазоны значений).
3) Из клинических протоколов подтаскивается дополнительный релевантный контекст 
3) LLM генерирует текстовые блоки отчёта по шаблону: диагноз/описание, таблица ключевых размеров, выводы, рекомендации.
4) Генерируется итоговый отчёт: HTML и PDF в формате, похожем на предоставленный пример.
5) Пользователь оставляет оценку отчёта (например, 1–5 + комментарий по качеству/ошибкам).
6) В базе сохраняются: входные файлы/ссылки, извлечённые измерения, версия промпта/модели, трассировки LLM, итоговый отчёт и оценки.

Функциональные требования:
Администратор:
- Управление шаблонами отчёта (версионирование промптов/шаблонов).
- Просмотр метрик качества: средняя оценка, частота ошибок валидации, время генерации
- Логирование и трассировка всех LLM-вызовов, возможность подключения llm as a Judge
- Обновление клинических протоколов лечения
Пользователь:
- Отправка запроса изображения+измерения+метаданные
- Просмотр списка отчетов
- Онлайн просмотр отчета
- Скачивание pdf отчета
- Оценка качества отчета

Технологический стек(рекомендуемый):
- Python 3.11+
- FastAPI (REST API)
- PostgreSQL + SQLAlchemy 2.0 / SQLModel
- Object storage (MinIO локально)
- Очередь задач (Celery + Redis/RabbitMQ)
- Langfuse и/или LangSmith
- React (Простая web-форма)
- Jinja2/pandoc (генерация отчетов)
